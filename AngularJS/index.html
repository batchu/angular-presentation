Users want Desktop style apps

  • HTML was originally designed to show static documents on the web.
  • But we want Desktop style apps. We get then by either
    • Abstracting away HTML, CSS, and/or JavaScript (GWT, Vaadin etc)
    • Manipulating the DOM (JQuery)
  • Neither of these address the root problem that HTML was not designed for dynamic views.
  • The web has enabled businesses to connect to its users with just a browser. And as we all know browsers can only understand HTML, CSS and Javascript.
  • But HTML was never meant to support development of interactive Desktop style apps like we do now. It was originally developed for displaying static content/documents. The HTML spec we still use today (version 4.0)was drafted in 1999. Now we are focusing on HTML 5 but there's still a lot that's not in there to help us in this regard.
  • So we use certain frameworks to deal with HTMLâ€™s shortcomings by either abstracting away from HTML, CSS, and/or JavaScript ( using GWT, Vaadin etc) or by manipulating the DOM (JQuery is a prime example for that).
  • Neither of these address the root problem that HTML is not designed for dynamic views. This is a problem angular tries to solve

DOM manipulation is evil!

What do we lose?

  • End to end testing capabilities
  • Maintainability
  • Reusability
  • Heavy mixing of business & UI logic through DOM manipulation makes
    • debugging a pain
    • It makes testing difficult
    • We cannot resue the code in other projects let alone somewhere else in the same project
    • It's very difficult to maintain

What is Angular Anyway?

“AngularJS is a structural meta framework for developing modern dynamic web applications”
  • AngularJS is a structural meta framework for developing modern dynamic web apps
  • Angular is what HTML would have been had it been designed for web applications.
  • It provides an End to End solution for development.
    • It is a Meta Framework
      • What I mean by "meta" is that it gives a lot of room for developers to expand on its existing functionality. We'll talk about it a bit later
    • It Works well with other js libraries nicely including JQuery (If we need to)
